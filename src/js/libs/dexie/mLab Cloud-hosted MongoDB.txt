way is via the |mongoimport| and |mongoexport| utilities. These allow
you to import and export JSON and CSV representations of your data. The

with binary dumps.

In this tab we provide pre-filled strings for the commands that we find

Conditional Operators

$gt  - greater than <value>, $lt - less than <value>,

$lte - less than or equal to <value>

e.g. { "size" : {"$lt": 3} }

$ne - not equals <value>

e.g. { "status" : {"$ne": "complete"} }

$mod - mod <divisor> is <value>

syntax: { <field> : {"$mod":[<divisor>, <value>]} }

e.g. { "size" : {"$mod": [10, 1]} }

$in - in <array>, $nin - not in <array>

e.g. { "size" : {"$in": [1, 2, 3]} }

Unique

A unique flag of |true| signifies a unique index. The |_id_| index is
always unique.

Parentheses indicate an implicit value. Documents representing index
definitions do not explicitly include |unique: false|, so the value of
unique is "(false)."


Sparse

A sparse flag of |true| signifies a sparse index, which only contains
documents with the indexed field. When sorting or filtering on a
sparsely indexed field, any documents that are missing that field will
be omitted from the query results.

Note that a sparse index can only have one field.

Parentheses indicate an implicit value. Documents representing index
definitions do not explicitly include |sparse: false|, so the value of
sparse is "(false)."


Min / Max

If |min| and |max| are blank, the index assumes lattitude/longitude and
is thus configured for a -180 to 180 value range.

collection. Views describe how to turn each JSON document (with
potentially nested field values) into a flat set of fields for display
as a row in a table.

To illustrate, let's say we have a collection with documents that look
like this:

  { "firstName" : "Bob",

    "address" : {

  { "Last Name" : "lastName",
